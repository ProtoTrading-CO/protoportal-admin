# Apollo Education Record

Permanent graduation reports — one per capability sub-sprint.

Apollo is learning **subjects**, not loading plugins. See **`docs/APOLLO_CURRICULUM.md`** for the report card, maturity scale, and **graduation conversations** (business acceptance tests).

Each report records **what Apollo learned**, not just what code shipped. Two states per capability:

| State | Meaning |
|-------|---------|
| **Engineering** | Apollo knows how (code, tests, evidence pattern) |
| **Operational** | Apollo is allowed to prove it (live data, bridge, production verification) |

Education does not stall on infrastructure. Engineering graduates when the capability is built. Operational graduates when live verification passes.

**Graduation conversation** — a real business dialogue that must pass before a capability is marked 🟢 (see `APOLLO_CURRICULUM.md`).
