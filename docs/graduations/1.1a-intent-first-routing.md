## What Apollo learned

Apollo learned that **a product is a business object, not a search result**.

---

## Problem

1. Entity Registry ran **before** Intent Engine — `tell me about the best selling item today` misrouted to customer lookup.
2. Apollo Chat still used the **keyword index** for SKU queries — `8626100145` returned *"No products matched"* while Product Context + live ERP already worked.

## Fix

```
User → Intent → Entity Registry → Product Context → Query Engine → Truth
```

Product route runs **first** in chat — before experience paths and before keyword search.

Keyword index is **fallback only**.

ERP title resolution: products not on the website still resolve via `stmast_cache` / live ERP — the ERP is Truth; the website is one manifestation.
