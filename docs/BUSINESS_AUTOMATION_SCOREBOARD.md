# Apollo Business Automation Scoreboard

**Purpose:** Measure operational transformation, not engineering output  
**Owner:** Proto operations  
**Review cadence:** Weekly

## Completion rule

A process is replaced only when its owner can complete normal daily work
without returning to the old notebook, spreadsheet, printed sheet, diary, or
manual report.

Code shipped, APIs created, and screens completed do not count as replacement.
Production evidence from real work is required.

## Weekly review

For each item under active development:

1. Which manual task will disappear?
2. Who performs it today?
3. How much time does it consume?
4. What errors, delays, or risks does it create?
5. What human approvals remain mandatory?
6. What production evidence proves replacement?
7. Has the old process actually stopped?

## Product KPI

> **If we finish this sprint, what manual task will Gee never have to do again?**

The long-term measure is equally direct:

> **If Apollo disappeared tomorrow, what manual work would Proto need to resume?**
